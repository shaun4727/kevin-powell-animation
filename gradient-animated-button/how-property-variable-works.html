Learn @property in 3 Steps

Step 1: The Blueprint (Syntax)

Whenever you want to use @property, you have to write out its "blueprint". You must include all three of these rules, otherwise the browser will ignore it.

@property --my-variable { /* 1. SYNTAX: What type of data is this? Options: <color>, <angle>, <length>, <percentage>, <integer> */ syntax: '<color>'; /* 2. INITIAL VALUE: If nothing else is set, what is the default? */ initial-value: transparent; /* 3. INHERITS: Should child elements inherit this variable? (Usually false for performance) */ inherits: false; }

Step 2: Animating an <angle>

Historically, you could never animate a conic-gradient (a circular color wheel). But if we register a variable as an angle, we can animate that variable from 0deg to 360deg, and the gradient will spin!

I am spinning using pure CSS!

@property --gradient-angle { syntax: '<angle>'; initial-value: 0deg; inherits: false; } /* The animation rule */ @keyframes spin { 0% { --gradient-angle: 0deg; } 100% { --gradient-angle: 360deg; } }

Step 3: Animating an <integer>

Because the browser knows our variable is a number, we can actually tell CSS to count from 0 to 100 over 5 seconds. We don't even need JavaScript!


@property --count-number { syntax: '<integer>'; initial-value: 0; inherits: false; } @keyframes count-up { 0% { --count-number: 0; } 100% { --count-number: 100; } }
